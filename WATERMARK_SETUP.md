# إعداد ميزة إضافة شعار الموقع على ملفات PDF

## 🎯 نظرة عامة

تم تفعيل ميزة إضافة شعار الموقع تلقائياً على الصفحة الأولى من كل ملف PDF يتم رفعه. الشعار يظهر في الزاوية العلوية اليمنى بشفافية 85%.

## ✅ التفعيل التلقائي

الميزة تعمل تلقائياً الآن! لا حاجة لأي خطوات يدوية.

### كيف تعمل:

1. **عند رفع كتاب PDF**: يتم استدعاء Edge Function تلقائياً
2. **تحميل الشعار**: يتم تحميل الشعار من `https://kotobi.xyz/kotobi-watermark-logo.png`
3. **إضافة الشعار**: يتم إضافة الشعار على الصفحة الأولى من PDF
4. **رفع الملف المعدل**: يتم رفع PDF مع الشعار إلى Storage
5. **حذف الأصلي**: يتم حذف الملف الأصلي (بدون شعار)

## 🔍 التحقق من عمل الميزة

1. ارفع كتاب PDF من صفحة "رفع كتاب"
2. انتظر حتى يكتمل الرفع
3. افتح الكتاب من صفحة القراءة
4. تحقق من وجود شعار الموقع في الزاوية العلوية اليمنى من الصفحة الأولى

## 🛠️ التخصيص

### تغيير موقع الشعار

في ملف `supabase/functions/add-pdf-watermark/index.ts`:

```typescript
// السطر 84-91
const margin = 20  // تغيير الهامش
firstPage.drawImage(logoImage, {
  x: width - logoWidth - margin,  // موقع أفقي
  y: height - logoHeight - margin, // موقع عمودي
  width: logoWidth,
  height: logoHeight,
  opacity: 0.85, // الشفافية (0.0 - 1.0)
})
```

### تغيير حجم الشعار

في نفس الملف (السطر 80):

```typescript
const logoWidth = 120  // تغيير العرض (بالبكسل)
```

## 🐛 استكشاف الأخطاء

### الشعار لا يظهر؟

1. **تحقق من Console Logs**:
   - افتح أدوات المطور (F12)
   - ابحث عن "🎨 بدء إضافة الشعار"
   - تحقق من عدم وجود أخطاء

2. **تحقق من Edge Function Logs**:
   ```bash
   # في Supabase Dashboard
   Edge Functions > add-pdf-watermark > Logs
   ```

3. **تحقق من وجود الشعار**:
   - تأكد من وجود `public/kotobi-watermark-logo.png`
   - الوصول إليه عبر: `https://kotobi.xyz/kotobi-watermark-logo.png`

### الملف يرفع ولكن بدون شعار؟

هذا سلوك طبيعي إذا:
- فشل تحميل الشعار من URL
- حدث خطأ في معالجة PDF
- الملف ليس PDF

في هذه الحالات، يتم رفع الملف الأصلي بدون شعار لضمان عدم فشل العملية بالكامل.

## 💡 ملاحظات مهمة
